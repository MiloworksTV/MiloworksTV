# site/

The public-facing static site for **MiloWorks Studios** — currently
just a single landing page that hosts the **Milo** show and points
visitors at the open-source [`miloworks`](../README.md) engine.

Deployed to [**miloworks.tv**](https://miloworks.tv).

## Files

```
site/
├── index.html        # the whole site (one file, no build step)
├── episodes.json     # generated by `miloworks publish` — DO NOT edit by hand
├── videos/           # generated mp4s + JPG posters — gitignored
└── README.md         # this file
```

`index.html` is dependency-free at runtime apart from Alpine.js and a
Google Font, both loaded from public CDNs. No bundler, no Node, no
build step. Drop the `site/` folder onto any static host and it works.

## Updating the site

After you've rendered new episodes (`miloworks run …` and
`miloworks stitch …`), run:

```sh
cd /path/to/miloworks
miloworks --project examples/milo publish \
  --site site \
  --name "Milo" \
  --tagline "A daily 60-second silent cartoon. One year. 365 episodes. Every frame chained to the last." \
  --description "$(cat <<'EOF'
A new episode every day for one year, all chained together as one
unbroken plano-secuencia: each shot picks up exactly where the last
one ended.

Milo himself is a small alien blob who shapeshifts according to
whatever feeling he can't yet name. He's brand new on Earth. Pip is
his quietly-patient owl friend who's been here forever.

The show is silent — ambient sound only, no dialogue — scored with
piano. Soft, sweet, watchable when you're tired. A small ritual you
share with the internet at the end of every day.
EOF
)"
```

That command:

1. Walks `examples/milo/episodes/`, picks every episode that has a
   `output/final.mp4`, copies the mp4 + extracts a JPG poster into
   `site/videos/`.
2. Writes `site/episodes.json` with show metadata + an episode list
   sorted newest-first.

`index.html` reads `episodes.json` at runtime via `fetch()` and
re-renders. So `miloworks publish` is the only command you need
between rendering a new episode and the site reflecting it.

## Deploying

Any static host works because everything is static after publish:

### Cloudflare Pages (recommended for video bandwidth)

```sh
# wrangler pages deploy site/
```

Pages has unmetered egress on the free tier, which matters once you
have hundreds of mp4s.

### Vercel

```sh
vercel deploy site/
```

Free tier has 100 GB/month egress — fine for low traffic, watch this
once the show grows.

### GitHub Pages

Push `site/` to a `gh-pages` branch (or use a workflow). Egress is
generous. Be aware GitHub serves with no `Range` request support on
some paths, which can cause large mp4s to buffer poorly.

### Just an S3 bucket

Whatever you've got. The site is six files (HTML + JSON + N mp4s).

## Hosting the videos elsewhere

The default is to ship `final.mp4` files alongside the site. If you'd
rather host videos on a CDN (R2, Bunny, YouTube embed, etc.), edit
`episodes.json` after publish and rewrite each episode's `video` field
to point wherever. The page only cares that `video` is a URL the
browser can load.

## Customizing

The whole site is one HTML file. Open it, change the colors at the
top, swap the font, rewrite the copy, ship it. No build step.

If you fork MiloWorks and run your own show, replace the show name
and copy in the `miloworks publish` command above and you're done —
this site becomes yours.
